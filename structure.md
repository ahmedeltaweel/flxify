## Architecture (app.js sections)

app.js is generated by `node build_app.js` — do not edit directly.

```
app.js (IIFE, self-contained bundle)
├── 1. Module System         # require('@flxify/name') shim, flxifyModules registry
├── 2. Register Lib Modules  # 7 modules inlined via registerModule() calls
├── 3. Script Registry       # 80 scripts embedded as template literals in SCRIPT_SOURCES[]
├── 4. Toast System           # showToast(message, type) - auto-dismiss notifications
├── 5. BoopState Class        # Script execution context (text, selection, fullText, postError, postInfo, insert)
├── 6. Script Executor        # executeScript() - new Function() wrapper, CodeMirror read/write
├── 7. Command Palette        # Show/hide, fuzzy search (name 0.9, tags 0.6, desc 0.2), keyboard nav
├── 8. Event Listeners        # Cmd/Ctrl+B toggle, search input, arrow keys, Enter, Escape
└── 9. Initialize             # parseScripts() (synchronous) → console.log
```

## Script API (BoopState)

Every script receives a `state` object with:

| Property/Method | Type | Description |
|-----------------|------|-------------|
| `state.text` | getter/setter | Selection if exists, else fullText (smart property) |
| `state.selection` | getter/setter | Currently selected text (null if none) |
| `state.fullText` | getter/setter | Entire editor content |
| `state.isSelection` | boolean | Whether text is selected |
| `state.postError(msg)` | method | Show red error toast |
| `state.postInfo(msg)` | method | Show blue info toast |
| `state.insert(text)` | method | Insert at cursor position |

## Module System

Scripts use `require('@flxify/moduleName')` to import libraries.
Modules are wrapped in a CommonJS sandbox and cached in `flxifyModules`.

## Build & Loading

**Build step:** `node build_app.js` reads sources from `web-scripts/` and `web-scripts/lib/`, then generates a self-contained `app.js` with everything inlined. No runtime `fetch()` calls — works with `file://` URLs.

**Runtime loading order (in generated app.js):**
1. `registerModule()` calls inline all 7 lib modules (synchronous)
2. `SCRIPT_SOURCES[]` contains all 80 scripts as template literals
3. `parseScripts()` parses metadata and populates the `scripts[]` array (synchronous)
4. Scripts sorted alphabetically by name

**Source of truth for edits:**
- `web-scripts/*.js` — 8 Flxify scripts
- `web-scripts/lib/*.js` — 7 library modules
- `build_app.js` — app runtime code template + build logic

## Tech Stack

- **Editor:** CodeMirror 6 via esm.sh CDN (one-dark theme, auto language detection)
- **Languages:** JSON, JavaScript, HTML, CSS, XML, SQL (auto-detected on content change, debounced 500ms)
- **Styling:** Custom CSS, no framework, flexbox layout
- **Build:** `node build_app.js` generates self-contained app.js
- **Works offline** from `file://` URLs (except CodeMirror CDN on first load)
